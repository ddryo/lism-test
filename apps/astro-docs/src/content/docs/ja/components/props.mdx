[Lismコンポーネント](/docs/components/overview)では、専用のプロパティ("Lism Props" と呼びます)が受け取れるようになっています。

<Preview>
	<PreviewTitle>Lism Propsを指定する例</PreviewTitle>
	<PreviewCode>
	```jsx
	<Box c="black" bgc="white" p={20} fz="xl">...</Box>
	```
	</PreviewCode>
</Preview>

{/* <Note heading="Lism Propsの重要なポイント" preset="point">
- プロパティと値の組み合わせによってはユーティリティクラス化されて出力される。
- ユーティリティクラス化されない時の挙動が数パターンある。
- ブレイクポイントで値を切り替えることができるものがある。
</Note> */}

{/* また、Lismコンポーネント以外では使用できませんので注意してください。 */}

## 対応トークンとユーティリティクラスでの出力について

各プロパティごとに、指定した値によってはユーティリティクラスとして出力される場合があります。

<Preview>
	<PreviewCode>
	```jsx
	<Box p={20} shadow="2" ta="center" fz="xl">...</Box>
	```
	</PreviewCode>
</Preview>
出力結果は次のようになります。

```jsx
<div class="l--box -p:20 -bxsh:2 -ta:l">...</div>
```

具体的にどの値をユーティリティ化するかどうかの詳細は`/src/config/presets`,`/src/config/utilities`で定義しています。

utilities側で定義しているものは、キーと省略形のどちらを指定してもユーティリティクラスとして出力されます。

たとえば、`text-align:center`用の `-ta:c` クラスは、以下のどちらの書き方でも出力されます。
```jsx
<Box ta="center">...</Box>
<Box ta="c">...</Box>
```


### ユーティリティクラス化されない時の出力パターンについて

ユーティリティクラスとして出力されない時、3パターンの出力方式に分かれます。

{/* - memo...
- 1のメリット
    - 何かしらそのpropを持つかどうかがクラスで判定できる。
	- 子要素に変数の受け渡しをすることで、親では何も適用せずにinheritさせるみたいなことができる。
    - 優先度的な問題が多少緩和できる。（強力なインラインセットは変数でしかない）
    - →HTML側の文字数が少なくなる(ほんの少し)
- 1,2の判断基準どうする？
	- utility以外を使うことが珍しいか（自由な数値で値をとれるか）どうか?
	- ブレイクポイントで切り替えるかどうかで判断すればいいのでは？ */}

#### 1. `-prop:`クラスと`--prop`変数の組み合わせで出力される


<Preview>
	<PreviewCode label="例" slot="tab">
	```jsx
	<Box mb={40} fz='2em'>...</Box>
	```
	</PreviewCode>
	<PreviewCode label="出力" slot="tab">
	```jsx
	<div class="l--box -mb: -fz:" style="--mb:var(--s--40);--fz:2em">...</div>
	```
	</PreviewCode>
</Preview>


- ユーティリティクラスが多く`[class*='-prop:']`にすると文字数の省略になりそうなもの
- `-{prop}:`クラスによって何か値を指定していることが分かったほうが便利なケースがありそうなプロパティ

#### `-prop:`クラスだけを出力する方法

このパターン1での出力がされるLism Propsに対して `"-"` または `true` を指定することで、`.-{prop}:` クラスだけの出力が可能です。

CSSファイル側で具体的な値を指定したい場合などに活用できたり、一部、`-{prop}:`クラスがスタイルを持っている特殊なケースもあります。

<Preview>
	<PreviewCode label="例" slot="tab">
	```jsx
	<Box bd w="-" p={['-', '-']}>...</Box>
	```
	</PreviewCode>
	<PreviewCode label="出力" slot="tab">
	```jsx
	<div class="l--box -bd: -w: -p: -p@sm">...</div>
	```
	</PreviewCode>
</Preview>



このように、`-{prop}:`クラスだけ出力されてCSS変数は何も出力されません。（パターン2で出力される`opacity`に関しては、`-opacity:`クラスが出力されていないことにも注目してください。）


#### 2. 単純にstyleとして出力されるもの


<Preview>
	<PreviewCode label="例" slot="tab">
	```jsx
	<Box z="10" opacity="0.5">...</Box>
	```
	</PreviewCode>
	<PreviewCode label="出力" slot="tab">
	```jsx
	<div class="l--box" style="z-index:10;opacity:0.5">...</div>
	```
	</PreviewCode>
</Preview>


#### 3. `--prop`だけ出力されるもの

このケースが一番稀です。

<Preview>
	<PreviewCode label="例" slot="tab">
	```jsx
	<Box bd bdc="#000" bdw="2px">...</Box>
	```
	</PreviewCode>
	<PreviewCode label="出力" slot="tab">
	```jsx
	<div class="l--box bd:" style="--bdc:#000;--bdw:2px">...</div>
	```
	</PreviewCode>
</Preview>


- `--bdc`,`--bdw`,`--bds` のborder系
- `--keycolor` など、利用する場面が限定されているもの。


## ユーティリティクラスとして強制的に出力する方法

`u:`からはじめると、その後ろの文字列がユーティリティクラス名として出力されます。

```jsx
<Box p='u:hoge'>...</Box>

↓

<div class="l--box -p:hoge">...</div>
```



## Responsive Props

Lism Propsでは、次の2通りの指定方法によって、ブレイクポイント指定用のユーティリティクラスとカスタムプロパティのペアで出力されます。

1. 配列で順番に指定（ _ → sm → md → ...）
2. オブジェクトで直接ブレイクポイントを指定

<Preview>
	<PreviewCode label="配列形式" slot="tab">
	```jsx
	<Box p={[20, 30, '5rem']}>
		...
	</Box>
	```
	</PreviewCode>
	<PreviewCode label="オブジェクト形式" slot="tab">
	```jsx
	<Box p={{ _: 20, sm: 30, md: '5rem' }}>
		...
	</Box>
	```
	</PreviewCode>
	<PreviewCode label="出力" slot="tab">
	```jsx
	<div class="l--box -p:20 -p@sm -p@md" style="--p--sm:var(--s--30);--p--md:5rem">
		...
	</div>
	```
	</PreviewCode>
</Preview>

<Alert preset="info">
どちらも最終的に出力されるHTMLは同じになり、そのHTMLとCSSを直接記載するだけでもブレイクポイントで値を切り替えることが可能です。
</Alert>

<Alert preset="alert">
CSS側でクラスが用意されていないと実際に値は切り替わりません。
</Alert>

途中のポイントをスキップする場合は次のようになります。

<Preview>
	<PreviewTitle>例2:`@md`の値のみ指定</PreviewTitle>
	<PreviewCode label="配列形式" slot="tab">
	```jsx
	<Box p={[null, null, 40]}>
		...
	</Box>
	```
	</PreviewCode>
	<PreviewCode label="オブジェクト形式" slot="tab">
	```jsx
	<Box p={{ md: 40 }}>
		...
	</Box>
	```
	</PreviewCode>
	<PreviewCode label="出力" slot="tab">
	```jsx
	<div class="l--box -p@md" style="--p--md:var(--s--40)">
		...
	</div>
	```
	</PreviewCode>
</Preview>
