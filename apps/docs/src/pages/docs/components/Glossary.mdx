# Glossary

用語リスト用のブロックモジュール。説明リスト。

- クラス名: `b--glossary`

どちかがよいか検討中
1.  `GlossaryItem`を並べるパターン
  - CSSで柔軟にレイアウトを組めるように予めdivで囲んだ状態で固定する。
  - `term`と`children`で用語と説明を指定するだけでOK.(使うの簡単)
  - コンポーネントのネストは少なくてすむ。
  - ただし、dt,dd それぞれに対してprops当てることができない。(別途それらのpropを用意すればいけるが...)
  - シンプルな用語リストとしての想定だけであればこれでいい。
2. `GlossaryTerm`,`GlossaryDesctiprion`を使ってもらうパターン
  - それぞれにpropsを当てることができるので柔軟性は増すが、少し使うのが面倒になる。
  - GlossaryItemで囲むと予めstackで配置するようにする？

それぞれにpropsを当てる = 普通にLismコンポーネントで自作コンポーネントとして組み直す方がいいので、①がいいか。


- `<Glossary>`
	- `<GlossaryItem>`
- Lismクラス: `.b--glossary`
	- `.b--glossary__item`
		- `.b--glossary__dt`
		- `.b--glossary__dd`

## Import

```js
import { Glossary, GlossaryItem } from '@lism/core';
```
