# コンセプト

"**Lism**" は、WEBサイト制作を効率化するためのフレームワークです。  
独自の**CSS設計理論**を元にした**Reactコンポーネントライブラリ**を提供します。

WEBサイトで一般的によく使われるレイアウト・デザイン要素を細分化し、汎用的に使い回せるようになっています。

を目標にしており、以下の3点を軸としてHTML&CSS設計理論を組み立てています。

	- トークン
	- ユーティリティクラス

これらを活用することで、**少量のCSSで柔軟なデザインを組める**ようになっています。

[Every Layout](https://every-layout.dev/) + [TailwindCSS](https://tailwindcss.com/) + [Bootstrap](https://getbootstrap.com/) を 混ぜ合わせたようなイメージです。

## Concept 1. あくまで HTML & CSS である。

Lismは、あくまでも**HTMLとCSSの設計**が主体です。

<Alert preset="point">
なので、CSSを読み込むだけで素のHTMLサイトで活用することもできます。
</Alert>

<Text mbs="20" fz="s">（一部、アコーディオンやタブの動作に関するJavaScriptが付属しています。）</Text>

ただし、配布している専用のReactコンポーネントを使ったほうがより手軽に扱えるようにはなるため、ViteやNext.js、Astro での活用を推奨しています。

(素のHTMLでもLism設計を活用しやすいように、Reactベースだけでなくweb components としても配布したいと思っています)


## Concept 2. 全てを利用しても軽量である。

素のHTMLで使うことができるユーティリティファーストで便利なフレームワークは、Lism以外にもたくさんあります。

しかし、それらを使っていてどうしても問題となるのは、**必要のないコードを大量に読み込んでしまうこと**です。

ビルド環境を工夫できる場合は何を使っても無駄を省くことができるので良いですが、そうでない場合は容易されているユーティリティクラスをすべて読み込む必要があります。


<Alert preset="point">
そこで、Lismは**用意しているすべてのCSSを読み込んでも気にならないCSS量**に抑えることを意識しました。
</Alert>

{/* また、完全なユーティリティファースト設計ではないため、レイアウトモジュールに関するCSSについては分離して必要なページでのみ読み込むこともできます。 */}

これにより、素のHTMLサイトや**WordPressサイト**にも活用しやすいようになっています。


## Concept 3. なのに、柔軟である。

ユーティリティファーストなフレームワークで一番課題となるのが、レスポンシブ対応です。


<Alert keycolor="blue" icon={SmileyXEyes}>
複数のブレイクポイントに対応したクラスを用意して柔軟性を高めようとすると、CSSの量が一気に肥大化してしまいます。
</Alert>

メディアクエリ等を使用しない素晴らしいCSS設計を採用してもいいかもしれませんが、実際のクライアントワークではどうしてもブレイクポイントでの切り替えが必要な場面が多いのが現状です。

そこでLismは、「CSS変数とユーティリティクラスの組み合わせ」を考えました。

<Alert keycolor="green" icon={ThumbsUp}>
CSS量を抑えつつ、**クラスと変数の組み合わせだけで簡単にブレイクポイントごとの指定ができる**ようになっています。
</Alert>


## その他の特徴


### コンテナクエリの採用

メディアクエリではなく、コンテナクエリを採用しています。


### セレクタ優先度の配慮

- `@layer`の採用
- `:where()`, `:is()` の採用
- `!important`をできるだけ使わない


### レンダリング負荷の低いセレクタを避ける

- `:has()` は使わない
- 子孫セレクタ（`.foo .bar`）を極力使わない
- ユニバーサルセレクタを極力使わない


### ダークモードの実装も簡単

カラーテーマの切り替えも、ルートのクラス名に合わせてトークンの切り替え(CSS変数の上書き)をするだけで実装できます。



### WordPressフレンドリー

Lism は、WordPressにも導入しやすいように設計されており、Lism設計に基づいたブロックテーマも開発予定です。

- 軽量
- WordPress側のクラス名と衝突しない命名規則
- Tokenを活用する点が、`theme.json` 関連機能との親和性が高い。
- コンテンツ幅の制御手法などもWordPressに合わせた設計が可能。
- Reactベースのコンポーネントがあるため、ブロック化しやすい。
- Lismベースのブロックでコンテンツを管理すれば、WordPressをヘッドレスCMSとして使いやすくなる。(サイト全体のCSSとコンテンツに必要なCSSがLismに集約されるため)


### Purge(Tree shaking)しやすいCSSの設計

ユーティリティクラスの活用に少し癖がありますが、PurgeCSS などを使って超軽量にできるようになっています。
`.-prop@bp:`の形式で個別のプロパティごとに各ブレイクポイントに対応したクラスが1つずつあるのも、パージしやすくするためでもあります。

## 今こそ車輪の再発明から解き放たれる時です

似たようなCSS、似たようなHTML、似たようなコンポーネント、を何度も1から書いていくのはもうおしまいにしましょう。

柔軟性があり、軽量で、どんなプロジェクトにも導入できるフレームワークがここにあります。

Chakra, Tailwind, Bootstrap, Bulma, などのフレームワークも参考にしています。
