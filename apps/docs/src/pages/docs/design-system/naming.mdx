# Naming Rules

- BEMと、WordPressコア機能で使用されるクラスとの衝突がない命名規則を採用。

- 変数
	- ケバブケース?
	- `[--{scope}]--{prop-name}[--{token|bp}]`
	- １文字省略形は大文字 (`--maxW` とか

## モジュールの分類と命名規則

|分類|形式|例|
|---|---|---|
|レイアウトモジュール<br /><small class="-d:ib">coreモジュール</small>|`.l--{moduleName}`|`.l--box`, `.l--layer`|
|エレメントモジュール<br /><small class="-d:ib">coreモジュール</small>|`.e--{elemName}`|`.e--badge`,`.e--icon`|
|ブロックモジュール<br /><small class="-d:ib">l--,e--の組み合わせ</small>|`.b--{blockName}`|`.b--banner`, `.b--section`|

{/* |インライン書式 ? |`.i--{書式名}` or `.f--{書式名}`|`.i--marker` or `.f--marker`| */}

- `{}`の内側部分はキャメルケース.
    - 例: `.b--mediaText`

- 各モジュールの子要素は2つの`__`で連結する。
	- `.{prfx}--{moduleName}__{elementName}`
	- 例: `.b--section__body`


### モジュールの関係性

- .l--系モジュールがLismのコアとなるレイアウトプリミティブ要素です。
	- .l--の中にさらに .l--が入ってきたりします。.e--ももちろん入る。
- .e--系はデザインの最小単位となるような小さな要素です。(buttun, badge, icon, ...)
	- iconはさらに e-- の中にも入ってくるので、命名規則は要検討...
- .b--系は、Lismのコアモジュール(.l--,.e--)を組み合わせてより具体的な用途を想定したモジュールです。使わなくてもいいし、自分で好きに定義し直したり、増やしたりしてください。

## data属性

「特定のモジュールに関する機能的なバリエーション」にあたるものはクラスではなくdata属性で管理する。

- 例: `.l--frame[data-ratio="4/3"]`, `.e--badge[data-variant="xxx"]`など

### CSS変数名の命名規則

- ローカルスコープの変数は プレフィックスなしでよさそう？
	- 例: `--gap`
- 同じ`{}`の中身はキャメルケースで。


- 形式: `[--ls][--{scope}]--{propName}[--{modifier}]`
	- `--{scope}` 付きの例
		- --root--fz
		- --inner--flowGap？
	- global変数（ rootで定義はされるが、各要素で使用されるもの ）は `--ls`ではじめる。(`ls`は仮)
		- 例: `--ls--flowGap`, `--main`, `--space--50`
	- `--{modifier}` にくるものについて
		1. プリセット値
			- `--fz--s`, `--space--20`, `--main`
			- `--ls--fz:s`, `--ls--space:20` のように 本当は`:`を使いたいが、CSS変数名では使えない。
		2. メディアクエリ
			- `--gap--sm`,  `--p--xs`,
			- 本当は メディアクエリは`-@sm`とかで記述したいが、`@`はCSS変数名では使えない。

- `{値}`について
	- ~~小数点の . は _ で表記？~~ → `.`のままでCSS側で`\`使う
		- 例: line-height: 1.5 のユーティリティ → `.-lh:1.5`
	- ネガティブ値の表記
		- `prop:-val`で書ける


## 単語の省略について
- Emmet形式をベースにして省略している。
	- background → `bg`
	- background-color → `bgc`
	- \`-ta:c\`
	- ※ Emmet に存在しないものどうする問題あり。
- CSS変数名の`{対象}` もわかりやすいものは省略可能。
	- text → txt (--c--txt)
	- button → btn (--c--btn)
	- background → bg (--c--bg)
	- 1文字になるものはあまり省略しない.
		- ただし、超メジャーな c, w, h, p, m は 1文字 で利用している。
