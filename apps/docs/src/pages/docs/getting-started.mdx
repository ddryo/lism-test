## 特定の環境に合わせた コンポーネントの読み込み

### Next.js

Next.js の **app ルーティング**を使用している場合、`<Accordion>`や`<Tab>`を使うには`"use client"`の記述が必要になります。

`@loos/lism-next`パッケージを追加すると、それを省略することができます。  
<small>（`"use client"`が必要なものはすでに記述された状態でimportできるようになります。）</small>

```bash filename="Add package"
npm i @loos/lism-next
```

パッケージから直接コンポーネントのソースコードをimportできるようにするため、`next.config.js`の`transpilePackages`に`"@loos/lism-next"`を追加してください。

```js filename="next.config.js"
const nextConfig = {
	transpilePackages: ["@loos/lism-next"],
	// ...
};

module.exports = nextConfig;
```


これらの準備により、`<Accordion>`などの`"use client"`が必要なコンポーネントも簡単に利用できるようになります。（Boxなどの普通のコンポーネントも、`@loos/lism-next`から同時にimportできます。）

```js
import { Box, Accordion, AccordionHeader, AccordionBody } from '@loos/lism-next';

<Accordion>
	<AccordionHeader>accordion heading...</AccordionHeader>
	<AccordionBody>
		<Box>content...</Box>
	</AccordionBody>
</Accordion>
```



### Astro

一部のコンポーネントに対してAstroでの利用に合わせて調整したものを配布しています。<MemoBadge>準備中</MemoBadge>

```bash
npm i @loos/lism-astro
```

```js
import { Box, ... } from '@loos/lism-astro';
```

Astroで利用方法が変わるコンポーネントについては、各コンポーネントの解説ページで個別に説明していく予定です。
