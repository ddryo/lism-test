# コンセプト

Lism UI は、WEBサイトの骨組みを素早く構築するためのWebフレームワークであり、独自に考えたHTML&CSSの設計理論をベースにしています。

HTMLとCSSだけで構築するWEBサイトにも採用可能ですが、直書きでは扱いづらい設計になっているため、Lism設計に沿ったコンポーネント集をOSSで公開しています。

「骨組みをLismモジュールでサクッと組み立て、最後の仕上げだけCSSを書いて整える」というような使い方をイメージしています。


## プリミティブなレイアウトモジュール

Lismでは、WEBサイトで採用されがちな**一般的なデザインを細分化し、プリミティブなレイアウトモジュールとして再利用可能な形で予め定義**しています。

例えば、FlexやGridレイアウトはWEBサイトで頻繁に採用されますが、何も考えずにCSSを書くと同じようなCSSを何度も書くことになります。

そういったことがないように、**主要なレイアウトモジュールを予め定義して再利用していく**のがベースの考え方です。


## プリセット値の活用

タイポグラフィ、余白、カラー、シャドウなど、**デザインの中で重要なプロパティについては予め段階的なプリセット値を定義**し、それらを使い回す設計になっています。

できるだけ事前に定義するプリセット値を活用することで、サイトを通して一貫性のあるデザインを実現でき、必要なコード量を減らすことができます。


## UtilityクラスとCSS変数の活用

主要なCSSプロパティに対して、**専用のUtilityクラスとCSS変数**を用意しており、次のような特徴があります。

- `.-{prop}:{presetVal}` で 事前に定義したプリセット値を利用できる.(※1)
- `.-{prop}:` クラスで `--{prop}` 変数を受け取れる.
- `.-{prop}@{breakPoint}:` クラスで `--{prop}--{breakPoint}` 変数を受け取れる.(※2)


<Alert>
※1: CSS変数用のプリセット値が全てユーティリティクラス化されているわけではないことに注意してください。プリセット値の中で、さらにどれをユーティリティクラス化するかを設定することができます。  
※2: ブレークポイントに対応しているプロパティのみ。
</Alert>

### 例

一例として`padding` について紹介します。

- `.-p:20` で 事前に定義したCSS変数 `--space--20` を利用する。
- `.-p:` クラスで `--p` 変数を、`.-p@sm:` クラスで `--p--sm` 変数を利用できるようになる。

という仕組みになっています。

CSSがどうなっているかは次の通りです。

```scss
[class*="-p:"]{padding: var(--p) !important}
.-p\:0{--p: var(--space--0)}
.-p\:10{--p: var(--space--10)}
.-p\:20{--p: var(--space--20)}
//...

@include query.cq('sm'){
	.-p\@sm\:{padding: var(--p--sm) !important}
}
@include query.cq('md'){
	.-p\@md\:{padding: var(--p--md) !important}
}
// ...
```

このようなCSSにより、Lism UI ではクラスと変数を書くだけでレスポンシブなレイアウトも手軽に実現できます。
主要なレイアウトに関して、自分でコンテナクエリを書く必要はほとんどありません。

実際の活用例は次の通りです。

```html
<!-- ↓ プリセット値の活用 -->
<div class="-p:20">...</div>

<!-- ↓ CSS書かずにレスポンシブ対応 -->
<div
	class="-p: -p@sm: -p@md:"
	style="--p:20px;--p--sm:30px;--p--md:40px"
>...</div>
```

このようにするだけで、プリセット値を手軽に利用できるだけでなく、ブレークポイントごとの上書き指定も手軽に行なえます。

Lism UI で事前に用意している軽量なCSSだけでも、かなり柔軟なレイアウトが実現できるようになっています。

<Alert>
ただし、ブレイクポイントごとのUtilityクラスは定義していません。（ファイルサイズが増大するため）
</Alert>


## WordPressフレンドリー
Lism UI は、WordPressでも活用しやすいように設計されています。

Lism設計に基づいたブロックテーマも用意しています。（開発中）

- ビルド処理が必要ない（HTML,CSSで完結できる）ため、WordPressのテーマに採用可能。
- WordPressコア機能で使用されるクラスとの衝突がない命名規則を採用。
- `theme.json` 関連機能との相性を考慮したプリセット名になっています。
	- タイポグラフィ: S,M,L などの"Tシャツサイズ"表記を採用。 2S ~ 6L まで用意。
		- 例: `--fz--s`,`--fz--xl`,
	- 余白: 0,10,20,...100 の数値量での命名規則を採用。
		- 例: `--space--10`,`--space--50`,
- コンテンツ幅の制御手法などWordPressに合わせた設計が可能。
- Next.jsなどで使えるコンポーネント、公式のWordPressテーマがあるので、WordPressをヘッドレスCMSとして使う場合にも利用しやすい。


## モダンCSSの積極的な活用

- grid, CSS変数, コンテナクエリ, ...
- IEは考慮してない。
- \`:has()\` は使っていません。


## レンダリング負荷の低いセレクタ

- 子孫セレクタを極力使わない
- ユニバーサルセレクタを極力使わない

PurgeCSS などを使えば超軽量になります。
